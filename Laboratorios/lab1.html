Preguntas

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?
    • Internet es una red de computadoras alrededor del mundo, que se usa para acceder a la World Wide Web, que es una recopilación de información y páginas que se asienta sobre esta.
    • “En cierto modo internet equivaldría a la infraestructura —las carreteras de países de todo el mundo— mientras que el contenido de las páginas web es lo que viaja sobre esa infraestructura —los autos, camiones, autobuses— para transportar información.” (BBC News Mundo, 2019)

  2. ¿Cuáles son las partes de un URL?
    • Protocolo: HTTP, HTTPS, entre otros.
    • Dominio: Nombre utilizado para identificar una o más direcciones IP.
    • Ruta: Especifica la ubicación del recurso o servidor.
    • Parámetros: Datos adicionales para identificar el recurso en el servidor.
    • (Aviani, 2019)
  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    • GET: Puede entenderse como “leer”, solicita datos de un recurso específico sin que este sea modificado de ninguna forma o cambie su estado.
    • HEAD: Es similar al GET, pero el resultado del método no contiene cuerpo, sólo encabezados, se utiliza para saber si el recurso existe.
    • POST: Envía datos al servidor para crear un nuevo recurso.
    • PUT: Actualiza un recurso existente usando el contenido, usando el contenido de la solicitud en el cuerpo.
    • PATCH: Aplica modificaciones parciales a un recurso.
    • DELETE: Elimina un recurso específico.
    • (Aviani, 2019)

  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
    • El método POST porque envía datos de información.

  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
    • El método GET pues está leyendo el contenido de la página para mostrar al usuario.

  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
    • Los códigos 2xx indican una solicitud exitosa, por lo que no ocurre ningún error. (Aviani, 2019)

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
    • Si, este error indica que el URL no se reconoce y por lo tanto no puede encontrar el recurso buscado, se trata de un enlace roto, dejando una mala experiencia al usuario, dando a entender que la página es deficiente y poco cuidada. (MDN contributors, 2019) (INBOUNDCYCLE, s.f.)

  8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
    • Si, porque indica un error interno del servidor, encontró una situación que no sabe manejar y por lo tanto debe ser corregida. (MDN contributors, 2019)

  9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
    • Que los antiguos atributos no son reconocidos en el entorno HTML5 puesto que hay funciones CSS maneja de una forma más eficiente. Entre los elementos de HTML4 que ya no están disponibles tenemos: type, axis, align, background, hspace, vspace, size, entre otros. (tutorialspoint, s.f.)

  10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
    • HTML5 está en continuo desarrollo a diferencia del 4 que tiene un estándar fijo, HTML5 cuenta con una sintaxis simplificada y fácil de leer, es compatible con HTML4, además añade los elementos de header y footer para una mejor anatomía de sitios web, section y article para códigos más estructurados y nuevas formas de insertar contenido multimedia con menos código. (Vargas González, 2012)

  11. ¿Qué componentes de estructura y estilo tiene una tabla?
    • table: contiene todo lo relacionado a la tabla.
    • caption: describe la tabla y su contenido.
    • thead: encabezado de las tablas, describe la información de cada columna.
    • tbody: contiene la información de la tabla.
    • tfoot: pie de la tabla.
    • tr: engloba cada fila de la tabla.
    • th: contiene la información de una celda individual de encabezado.
    • td: contiene la información de una celda individual.
    • colgroup: agrupa una o más columnas para dar formato.
    • col: define una columna única de información. (Nice, 2017)

  12. ¿Cuáles son los principales controles de una forma HTML5?
    • input: define el tipo de información que meterá el usuario.
    • type=””: especifica el tipo de campo, puede ser de texto, contraseña, fecha-hora, checkbox, entre otros.
    • value=””: valor del campo.
    • size=””: tamaño de caracterese del input.
    • step=””: número de intervalos legales para un input. (Nice, 2017)

  14. Sobre el ciclo de vida y desarrollo de los sistemas de información:
    • ¿Cuál es el ciclo de vida de los sistemas de información?
      • “Es un sistema ya sea automatizado o manual que engloba personas, máquinas y métodos organizados para recopilar, procesar, transmitir datos que representan información. Abarca la infraestructura, la organización, el personal y todos los componentes necesarios para la recopilación, procesamiento, almacenamiento, transmisión, visualización, diseminación y organización de la información.” (Redacción PowerData, 2019)
    • ¿Cuál es el ciclo de desarrollo de sistemas de información?
      • Es un activo corporativo esencial que se divide en 7 fases:
      1. Fase de planeación
      2. Fase de análisis
      3. Fase de diseño
      4. Fase de desarrollo
      5. Fase de integración y periodo de pruebas
      6. Fase de implementación
      7. Fase de mantenimiento (Redacción PowerData, 2019)
